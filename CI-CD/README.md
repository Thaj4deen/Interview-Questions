# **DEVOPS & CI/CD — SPOKEN VERSION**

1. **What is DevOps?**

    > DevOps is a practice that combines development and operations teams.
    > It focuses on automation and continuous delivery of software.
    > It helps to release applications faster and more reliably.
    > DevOps improves collaboration between teams.

##
2. **What is CI/CD pipeline?**

    > CI/CD pipeline is an automated process for software delivery.
    > Developers push code and the pipeline builds the application.
    > Then it runs tests automatically.
    > Finally, the application is deployed to servers.

##
3. **What is Continuous Integration?**

    > Continuous Integration means developers merge code frequently.
    > Each code change is automatically built and tested.
    > It helps to find bugs early.
    > It improves code quality and teamwork.

##
4. **Have you used GitHub Actions and Terraform?**

    > Yes, I used GitHub Actions for CI/CD automation.
    > It helped to build and deploy applications automatically.
    > I used Terraform for Infrastructure as Code.
    > It helped create Azure resources using scripts.
    > These tools improved automation and deployment speed.

##
5. **Difference between Continuous Delivery and Continuous Deployment?**

    > Continuous Delivery prepares the code for release.
    > But deployment to production requires manual approval.
    > Continuous Deployment automatically releases code to production.
    > No human approval is needed.

##
6. **What is a YAML pipeline?**
   
    > YAML pipeline is a pipeline written as code.
    > It is stored in the source code repository.
    > It allows version control and easy editing.
    > It is commonly used in Azure DevOps.

##
7. **Difference between YAML pipeline and Classic pipeline?**

    > YAML pipeline is configured using code.
    > Classic pipeline is configured using graphical interface.
    > YAML pipelines are easier to version control.
    > Classic pipelines are easier for beginners.

##
8. **What is a Multi-stage pipeline?**

    > Multi-stage pipeline has multiple stages like build, test, and deploy.
    > Each stage runs one after another.
    > It helps to manage complex workflows.
    > It improves release quality.

##
9. **Pipeline worked yesterday but failing today. How do you troubleshoot?**

    > First, I check the pipeline logs to find errors.
    > Then I verify recent code changes.
    > I check build agents and service connections.
    > Finally, I test the pipeline again.

##
10. **Pipeline is stuck in queue. What will you do?**

    > I check whether build agents are available.
    > If agents are offline, I restart them.
    > I verify agent pool configuration.
    > Then I retry the pipeline.

##
11. **How do you manage secrets in CI/CD pipelines?**

    > I store secrets in Azure Key Vault or secure variable groups.
    > I never hardcode passwords in code.
    > Pipelines access secrets securely during execution.
    > This improves application security.

##
12. **What CI/CD tools have you used?**

    > I have used Azure DevOps pipelines for automation.
    > I worked with GitHub Actions for workflows.
    > I also have experience with Jenkins.
    > These tools help automate build and deployment.

##
13. **Can we link YAML pipeline with Classic pipeline?**

    > Yes, we can link both pipelines.
    > YAML pipeline is code-based and stored in repository.
    > Classic pipeline is UI-based pipeline.
    > Sometimes YAML builds artifact and classic handles deployment.
    > But modern projects prefer full YAML pipelines.
